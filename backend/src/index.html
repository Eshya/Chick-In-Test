Both git rebase and git merge ar used to synchronise changes between branches. The difference between to command is git rebase will reivse the commit but keep clean organised history, while git merge will keep all the original commits and add ones automatically but gives out circuit board line like history.

Merging commits changes to another branch. It takes the contents of a source branch and integrates it with a target branch.
